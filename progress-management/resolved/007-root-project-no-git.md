# root-project/ が Git 管理されていない

## 発見日
2026-03-04

## 関連ステップ
プロジェクト全体

## カテゴリ
infrastructure

## 問題
`project/` 配下に `.git/` があるが、`root-project/` にはない。設定・ルール・仕様書がバージョン管理されておらず、誤操作で消失するリスクがある。

## 影響
- 設計ドキュメントの変更履歴が追えない
- 誤削除時にリカバリできない

## 提案
1. `root-project/` も別の private リポジトリで管理する
2. ローカルバックアップの仕組みを設ける
3. 重要なドキュメントは `deliverables/` → `project/docs/` へ転記する運用で公開リポジトリ側に残す

## 解決内容
提案1を採用。`root-project/` で `git init` を実施し、独立したローカル Git リポジトリとして管理を開始。
- `.gitignore` に `project/`（別リポジトリ）等を除外設定
- 初回コミットで全55ファイルを追加
- リモートリポジトリ（GitHub private）への接続は必要時に別途実施
