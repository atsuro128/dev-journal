# 並列実行時のファイル競合回避ルールの策定

## 発見日

中

## 発見経緯

## 関連ステップ
Step 4+5 以降（並列実行開始時）

## 問題

work-breakdown では複数セッションが同じファイル（openapi.yaml, screens.md 等）に書き込む設計になっている。並列実行時に複数の Claude Code セッションが同一ファイルを編集すると、git マージ競合が発生する。

## 影響

- 並列実行の成果が競合で失われるリスク
- マージ作業の手間が発生

## 提案

以下のいずれかの方式を検討:
- A: ファイルをセクション単位で分割し、各タスクが担当セクションのみ編集する
- B: 機能ごとに別ファイルで出力し、統合タスク（4+5-G）でマージする
- C: git worktree / branch を活用し、タスクごとにブランチを切って最後にマージする

Step 4+5 着手前までに方針を確定する。
