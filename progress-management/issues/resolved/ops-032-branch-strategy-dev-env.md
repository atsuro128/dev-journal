## 問題

並列開発を実施するための技術基盤（ブランチ戦略・CI/CD パイプライン・開発環境の複数メンバー対応）が未定義。現状の単一ブランチ運用では複数セッションが同時に作業するとファイル競合が発生する。また、Dev Container が単一インスタンス前提の設計であり、複数メンバーが同時に開発環境を利用する想定がない。さらに、CI/CD パイプライン設計が Step 7 Phase 1（実装フェーズ）に配置されているが、ブランチ保護・自動テスト・マージ前チェックは並列開発の前提条件であり、設計は実装開始前に完了すべきである。

## 影響

- 並列実行時に git マージ競合が頻発する
- 複数の Dev Container インスタンスでポート競合・DB 競合が発生する
- 並列実行の技術的前提が満たされず、マルチチーム運用に移行できない
- CI/CD 設計なしでは Step 6（テスト設計）の完了条件「CI で自動テストが走る」を満たせない

## 対応内容

以下を定義する成果物 `ai-dev-framework/rules/branching.md` を作成（Step 7 Phase 1 から前倒し）:

- ブランチモデル（main / develop / feature、タスク ID ベース命名）
- git worktree 運用方針（並列セッションの分離）
- タスク完了時のマージ順序・コンフリクト解決責任者
- CI/CD パイプライン設計（ステージ構成、トリガー条件、ブランチ保護ルール、マージ前チェック）
- Dev Container 複数インスタンス対応（ポート競合回避、DB 分離）

注: Step 7 Phase 1 には「設計済み CI パイプラインの実装」（GitHub Actions YAML 等）のみ残す。
