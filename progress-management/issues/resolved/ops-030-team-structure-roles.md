## 問題

project_steps.md は成果物の作成手順に特化しており、マルチチーム並列開発に必要なチーム体制の定義が存在しない。現状は「1人間 + 1AI（直列）」で運用しているが、ポートフォリオ目標は「複数の人間 + 各自の AI アシスタントが並列で開発する運用」を示すことであり、体制定義なしでは並列開発への移行判断ができない。

## 影響

- 並列実行時に誰が何を担当するか不明確
- AI が単独で判断してよい範囲が定義されておらず、過剰な確認か独断が発生する
- タスク割り当て方式が未定義のため、並列実行の運用設計ができない

## 対応内容

以下を定義する成果物 `ai-dev-framework/rules/team-structure.md` を作成:

- 人間ロール（PM / Tech Lead / Developer）の責務定義
- AI ロール（Worker / Reviewer / Coordinator）の責務と権限範囲
- 人間+AI ペアの構成モデル（1:1 か 1:N か）
- タスク割り当て方式（自己選択 / PM 割当 / 自動）
- 意思決定権限マトリクス（AI 単独可 / 人間承認必須の境界）

## 対応タイミング

## 解決内容

`.claude/rules/team-structure.md` を作成し、以下を定義:

- **ロール定義**: 人間（承認・判断）、指揮役（分配・品質ゲート・進捗管理）、サブエージェント 6体（設計フェーズ）
- **実行プロセス**: Phase 0（計画）→ Phase 1〜3（Wave 実行: worktree → PR → レビュー → マージ）→ Phase 4（統合 + 横断レビュー）→ Phase 5（指摘対応）
- **品質ゲート判定基準**: PASS / FIX / ESCALATE / PASS with NOTE
- **意思決定権限マトリクス**: 指揮役単独可 vs ユーザー承認必須の境界

AI ロールの詳細（15体の責務・モデル選定）は `dev-journal/ai-operations/subagent-design.md` に設計根拠として残置。
`.claude/rules/team-structure.md` は指揮役が実行時に従うルールとして簡潔に記述。

ops-026（レビュー粒度）もこのルールファイルに統合して解決。
