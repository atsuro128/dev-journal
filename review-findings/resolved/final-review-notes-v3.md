# 最終レビュー（残り4点）

## 1. `RPT-004` の「作成者（Member）」表現を修正
### 対象
- `04_business-rules.md`

### 指摘
`RPT-004` が「レポートは作成者（Member）に紐づく」となっているが、現仕様では Approver / Admin も自分のレポートを作成可能。

### 修正案
- `作成者（Member）` → `作成者（ユーザー）`
- または `所有者`

---

## 2. パスワードリセットメールと「メール通知スコープ外」の整理
### 対象
- `requirements.md`

### 指摘
MVP に `AUTH-F06 パスワードリセット` があり、リセットトークンのメール送信を前提としている一方、「やらないこと」に `メール通知` が入っているため、読み手が混乱しうる。

### 修正案
スコープ外の `メール通知` に注記を追加する。

#### 追記例
- `業務イベント通知メール（提出・承認・却下・支払完了）は対象外`
- `ただし、認証機能としてのパスワードリセットメールはMVP対象`

---

## 3. アクター分析の「承認者の割り当て」を現仕様に合わせる
### 対象
- `02_actor-analysis.md`

### 指摘
Admin の初期セットアップに「承認者の割り当て」とあるが、現仕様では MVP は担当者割当なし / submitted 全件承認型。

### 修正案
次のどちらかに修正する。

- `Approver ロールの付与（Phase 3で担当者割当を拡張予定）`
- `承認者の割り当て（Phase 3）`

---

## 4. `RBAC-F02` を「所有者ベース」に寄せる
### 対象
- `requirements.md`

### 指摘
`RBAC-F02` が「Member は自分が作成したリソースのみ操作可能」となっているが、現仕様では Approver / Admin も申請者として自分のレポートを持つ。

### 修正案
`Member は` をやめて、所有者ベースの表現に統一する。

#### 修正文例
- `申請系操作は所有者のみ可能`
- `Member / Approver / Admin は、自分が作成したリソースのみ操作可能`
