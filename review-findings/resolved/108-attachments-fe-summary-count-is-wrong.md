# 108: attachments.md の FE サマリー件数が実 ID 数と一致していない

## 指摘概要
`attachments.md` の FE サマリーが `90件` になっているが、同ファイルに定義されている `ATT-FE-001` から `ATT-FE-087` までのユニーク ID 数は 87 件である。  
レビュー観点で求められている「採番衝突なし、サマリー表との件数一致」を満たしておらず、テスト設計の進捗把握や実装漏れ確認に誤差が出る。

## 根拠
- `deliverables/docs/60_test/test_cases/attachments.md:724`
  - `**FE テストケース合計: 90件**`
- 同ファイル内の FE テスト ID は `ATT-FE-001` 〜 `ATT-FE-087` で欠番なし
- 実数確認:
  - `python` で `ATT-FE-\d+` を抽出するとユニーク件数は 87 件

## 判定
重大度: 低  
分類: テスト ID サマリー整合

## 修正方針案
- `attachments.md` の FE サマリー件数を実 ID 数に合わせて修正する
- 併せて優先度別件数（高 / 中）も再計算し、合計と一致することを確認する
- 今後同種のズレを防ぐため、サマリー更新時はユニーク ID 数を機械的に数えて確認する
