# 064: ui-guidelines.md に Step 5 正本が求めるテーマ/コンポーネント契約が不足している

## 指摘概要
Step 5 の正本は `ui-guidelines.md` に MUI テーマ設定のうちカラーパレット・タイポグラフィ・スペーシングを含め、さらにコンポーネント使用ガイドを「使用 OK / 非推奨 / カスタム必要」の区分で定義することを要求している。現行文書はカラーパレットとフォント、画面種別ごとの推奨コンポーネントはあるが、スペーシング契約と区分付きの使用ガイドがないため、フロント実装時の共通ルールとしては不足している。

## 根拠
- `ai-dev-framework/guide/work-breakdown/step5-detail-design.md:153-161`
  - `ui-guidelines.md` に「MUI テーマ設定（カラーパレット・タイポグラフィ・スペーシング）」および「コンポーネント使用ガイド（使用 OK / 非推奨 / カスタム必要の区分）」を含めると定義している
- `dev-journal/deliverables/docs/50_detail_design/ui-guidelines.md:33-61`
  - カラーパレットとフォントはあるが、スペーシング規約がない
- `dev-journal/deliverables/docs/50_detail_design/ui-guidelines.md:116-129`
  - コンポーネント一覧はあるが、使用 OK / 非推奨 / カスタム必要の区分になっていない

## 判定
- 重大度: 中
- 分類: 完了条件不足 / フロント実装契約不足

## 修正方針案
- `theme.spacing` の基準値、余白スケール、主要レイアウトで使う spacing ルールを追加する
- コンポーネント表を「使用 OK / 非推奨 / カスタム必要」に再編し、判断基準と代表例を明記する
- screens 側で参照すべき共通ルールがどこにあるかを明確にし、個別画面仕様への重複記述を避ける
