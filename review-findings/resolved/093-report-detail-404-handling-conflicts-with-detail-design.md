# 093: レポート詳細の 404 時挙動が詳細設計と矛盾している

## 指摘概要
`report-detail.md` の `ReportDetailPage` 責務は「レポートが存在しない場合は一覧にリダイレクト + トースト表示」としているが、画面詳細仕様では 404 はメインコンテンツ領域にエラーメッセージを表示し、一覧へのリンクを出す契約になっている。404 の UX が設計段階で二重定義になっており、実装者がどちらを採るべきか判断できない。

## 根拠
- `55_ui_component/screens/report-detail.md:68` では「レポートが存在しない場合は一覧にリダイレクト + トースト表示する」と記載されている。
- `50_detail_design/screens/report-detail.md:431-431` では `404 Not Found` の表示方法を「メインコンテンツに『指定されたデータが見つかりません。』」、挙動を「レポート一覧へのリンクを表示」と定義している。
- 同じ詳細設計でも `50_detail_design/screens/report-detail.md:378-379` はアクセス制御表でリダイレクト系の記述を含んでおり、Step 5.5 側で 404 表示契約を明示的に固定しないと下流で解釈が割れる。

## 判定
高 / 上流整合性不備

## 修正方針案
Step 5 の詳細設計の 404 契約に合わせて、`ReportDetailPage` の責務・状態遷移・対応表を「メインコンテンツ内の not found 表示 + 一覧リンク」に統一する。もし Step 5 側をリダイレクト方針に寄せるなら、先に詳細設計の記述を一本化してから Step 5.5 を合わせる。
