# コード内コメントの日本語化

## 発見日
2026-04-01

## カテゴリ
implementation

## 影響度
低

## 発見経緯
user-report

## 関連ステップ
Step 8（基盤構築）

## 問題
expense-saas/ 配下の全ソースコード（cmd/, internal/, frontend/）のコード内コメントが英語で書かれており、レビュワー（ユーザー）が内容を把握しづらい。

## 影響
- ユーザーが成果物を確認・理解する際の障壁になっている
- コードの保守性がユーザー視点で低下している

## 提案
- godoc 向けコメント（関数・型の直前のコメント）は英語のまま残す
- それ以外のコード内説明コメントを全て日本語化する
- 対象: `cmd/`, `internal/`, `frontend/` 配下の全ファイル
- 今後の実装時にもコード内コメントを日本語で書くルールの追加を検討する
