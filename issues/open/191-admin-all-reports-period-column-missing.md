## 問題

`SCR-ADMIN-002`（全レポート画面、Accounting / Admin で利用）の検索エリアには **対象期間フィルタ（期間：開始日 〜 終了日）**が存在するが、検索結果のレポートテーブルには **対象期間（period_start / period_end）カラムが無い**ため、フィルタが効いたかどうかを視覚的に確認できない。

### 設計の現状（`admin-all-reports.md` §5）

- **検索フィルタ**: ステータス / 申請者 / **期間（period_start ≥ from AND period_end ≤ to）**
- **テーブル列**: 申請者名 / タイトル / 合計金額 / ステータス / **提出日**（対象期間列なし）

### 業務シナリオ上の違和感

- UC-AC03（経理: 月次の経費全体の俯瞰）で「2026 年 4 月分の経費を確認したい」と思って期間フィルタに `2026-04-01 〜 2026-04-30` を入れると、フィルタは効く（period_start/period_end に対する絞り込み）が、テーブルに対象期間列がないため**結果が期間条件を満たしているか**を一覧上で確認できない
- 「提出日」列はあるが、これは対象期間とは別の概念（提出が遅れたレポートだと提出日は対象期間より後になる）

## 影響

- UAT-018 で発覚した違和感（UX 妥当性に関わる、NFR-UX-002 業務担当者の一読可能性）
- フィルタした結果の妥当性確認のためにユーザーがレポート詳細を開いて確認する必要があり、業務効率が低下
- ただし、機能としては設計通りに動作しており、ブロッカーではない

## 提案

### 案 A: テーブルに対象期間列を追加
- カラム例: 「対象期間」 = `YYYY/MM/DD - YYYY/MM/DD`（または `2026/04` 等の月表記）
- メリット: フィルタとの整合が取れる
- デメリット: 横幅増（既に 5 列ある）、レスポンシブで列削減対象になる可能性

### 案 B: 期間フィルタを「提出日範囲フィルタ」に変更
- フィルタ条件と表示列の整合が取れる
- ただし、業務上は「対象期間（経費が発生した月）」での絞り込みも有用
- 経費レポートの管理単位は対象期間が自然（要件 RPT-002）

### 案 C: 両方残す（提出日フィルタも追加、対象期間列も追加）
- 最も柔軟だが UI が複雑化

## 着手タイミング

**post-MVP**。MVP リリース後に UX 観点の改善として再検討する。
