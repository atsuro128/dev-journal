## 問題

ops-054 でテンプレートに責務境界ルールを追加したが、既存の成果物にはテンプレート導入前から存在する逸脱が残っている。codex レビューで全7ファイル NG と判定。

### architecture.md
- API エンドポイント詳細・レスポンス形式・ページネーション仕様（L309-389）がテンプレートの「機能別詳細設計を扱わない」に抵触
- 「扱わない内容」がテンプレートの最新版（代替案比較表・図表現全般の除外）を反映していない

### diagrams.md
- 「扱わない内容」がテンプレートの最新版を反映していない
- 状態遷移図・ローカル開発環境がテンプレートの役割外

### ADR-0003
- 設計仕様の要約（適用対象テーブル一覧、接続方式要点）が「扱わない内容」の境界上

### ADR-0005
- 詳細設計レベルの仕様が残っている（標準ログフィールド、メトリクス仕様、/health 契約、通知方式）

### 全7ファイル共通
- 「この文書の役割」テーブルがテンプレートの最新版を反映していない

## 影響

- 上流成果物が正しくないと下流の Step 5 以降が間違った前提で作られるリスクがある

## 提案

1. 各成果物の「この文書の役割」テーブルをテンプレートの最新版に合わせる
2. architecture.md の API 詳細を Step 5 に移動（参照リンクに差し替え）
3. ADR-0005 の詳細設計仕様を monitoring.md に移動
4. diagrams.md の状態遷移図・ローカル開発環境の扱いを整理
