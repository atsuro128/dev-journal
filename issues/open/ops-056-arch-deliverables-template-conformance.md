## 問題

ops-054 でテンプレートの責務境界ルールを整備し、成果物の一部を修正したが、以下の逸脱が残っている。

### architecture.md
- API エンドポイント詳細・レスポンス形式・ページネーション仕様（§5 API 設計方針）がテンプレートの「機能別詳細設計を扱わない」に抵触
- 「この文書の役割」テーブルの「扱わない内容」に、テンプレートで追加した除外ルール（代替案比較表、図表現全般）が反映されていない

### diagrams.md
- 「この文書の役割」テーブルの「扱わない内容」に、テンプレートで追加した「図の正本は本書のみ。architecture.md には図を再掲しない」が反映されていない
- 状態遷移図（§5）・ローカル開発環境（§7）がテンプレートの正本情報（システム構成図、リクエスト処理フロー、デプロイ/配信概要図）の範囲外

### ADR-0005
- 詳細設計レベルの仕様が残っている（標準ログフィールド定義、メトリクス仕様、/health エンドポイント契約、通知方式）。ADR の責務（判断根拠の記録）を超えており、monitoring.md の責務

## 影響

- 上流成果物の責務逸脱が下流（Step 5 以降）に間違った前提を与えるリスクがある

## 提案

1. architecture.md: 「この文書の役割」テーブルにテンプレートの除外ルールを反映。API 詳細は Step 5 の openapi.yaml / api_design.md への参照に差し替え
2. diagrams.md: 「この文書の役割」テーブルを更新。状態遷移図・ローカル開発環境の扱いを整理（削除 or テンプレートの正本情報を拡張）
3. ADR-0005: 詳細設計仕様を monitoring.md に移動し、ADR は判断根拠 + forward reference に整理
