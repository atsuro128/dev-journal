# 非機能要件のテストカバレッジが不十分

## 発見日
2026-04-02

## カテゴリ
testing

## 影響度
中

## 発見経緯
user-report

## 関連ステップ
Step 6（テスト設計）、Step 11（システムテスト・UAT）

## 問題

非機能要件（NFR-PERF, NFR-AVAIL）を定義しているが、それを検証するテストが MVP スコープ外とされており、実施計画が定義されていない。

### テストが不足している非機能要件

| 要件ID | 要件 | 現状 |
|--------|------|------|
| NFR-PERF-001 | API レスポンス p95 500ms 以下 | 20回のスモークテストのみ。本格的な負荷テストなし |
| NFR-PERF-003 | 同時接続 100ユーザー | テストなし（トレーサビリティに「MVP スコープ外」と明記） |
| NFR-AVAIL-001 | 稼働率 99.5% | テストなし（本番環境でのみ計測可能） |

### 検証できていないリスク

- 同時アクセスでコネクションプールが詰まるか
- RLS の set_config が高負荷時に正しく動作するか
- レートリミッターが高負荷時にメモリを圧迫しないか

## 影響

- 非機能要件を定義しながら検証していないため、要件を満たしているか不明
- 本番リリース後に性能問題が発覚するリスク

## 提案

1. Step 11（システムテスト・UAT）に負荷テストの実施計画を追加する
   - ツール: k6 または wrk
   - シナリオ: 100ユーザー同時接続でのレスポンスタイム計測
   - ボトルネック分析: コネクションプール、DB クエリ、メモリ使用量
2. テスト結果をポートフォリオの実績データとして活用する
