## 問題
workflow.md にセッション中の作業フロー（PR コメント投稿、codex レビュー前の fetch、押し返し手順など）が全て記載されているが、指揮役が長い手順書を参照しながら作業すると見落としが発生する。実際に codex への押し返し時に PR コメントを投稿し忘れるミスが発生した。

## 影響
手順の見落としによるプロセス違反が繰り返される。特にレビューサイクル（押し返し・再レビュー）のような条件分岐を含む手順で発生しやすい。

## 提案
workflow.md の手順をスキル（`.claude/skills/`）に細分化し、スキル実行時に手順が自動的に展開される構造にする。例:
- `/codex-review` スキルに「押し返し時は PR コメント投稿 → 再レビュー」のステップを追加
- PR フローの各段階（CI 監視、レビュー対応、マージ前確認）をスキル化
- workflow.md は全体像の俯瞰に留め、詳細手順はスキルに委譲
