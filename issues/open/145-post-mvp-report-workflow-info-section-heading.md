## 問題

現状の実装では `ReportInfoCard` の ReportWorkflowInfo セクション（提出・承認・却下・支払の履歴情報）が `<Paper variant="outlined">` で枠線のみ表示され、枠内にセクション見出しが付いていない。対してレポート詳細画面内の「明細一覧 (N件)」セクションには明示的な見出しがあり、情報粒度が同レベルなのに見出し有無に一貫性がない。

### 現状

ファイル: `expense-saas/frontend/src/pages/reports/ReportInfoCard.tsx:83-89`

```tsx
{hasWorkflowInfo && (
  <Paper
    variant="outlined"
    sx={{ mt: 2, p: 2 }}
    data-testid="report-workflow-info"
  >
    {/* 提出日 / 承認情報 / 却下情報 / 支払情報 */}
  </Paper>
)}
```

### 設計書

- `50_detail_design/screens/report-detail.md:56` レイアウトモック内: `| | --- 承認/却下/支払情報（該当状態の場合のみ）--- |      |` — ASCII 区切り風だが UI 見出し指定かは曖昧
- `55_ui_component/screens/report-detail.md:130-159` `ReportWorkflowInfo`: 責務の記述のみ、見出し表示指示なし

## 提案

見出しを追加するが、**具体的な見出し文言は本 issue 対応時に決定**する。以下の候補を比較検討する。

### 見出し文言候補

| 候補 | ニュアンス | 利点 | 欠点 |
|------|---------|------|------|
| 処理履歴 | 提出 → 承認 → 支払の時系列を示唆 | 簡潔（漢字 4 文字）。中立（却下・承認・支払を包含）。業務ユーザー向け | やや抽象的で「何の履歴か」が初見で伝わりにくい可能性 |
| ステータス履歴 | 状態遷移の記録 | 意味明確、技術用語でもない | やや硬め、文字数が多い（6 文字） |
| 承認・支払情報 | 具体的 | 内容を直接説明 | 「却下」を暗黙にするため、却下状態で違和感が出る可能性 |
| ワークフロー情報 | 処理フローの情報 | 設計書 L56 の ASCII 区切りの記述と近い | 技術用語で業務ユーザーに硬い |
| 進捗情報 | 進み具合の記録 | 業務寄り、分かりやすい | 「進捗」は完了ステップ数などを連想させ、時系列履歴と違う語感 |

### 判断観点

1. **業務ユーザーへの分かりやすさ**: 技術用語（ワークフロー / ステータス）を避ける
2. **却下状態での違和感の有無**: 「承認・支払情報」は却下時に矛盾するため却下時も中立な表現を選ぶ
3. **文字数**: スマホ幅（375px）でも無理なく収まる（漢字 4-6 文字程度が目安）
4. **他セクションとの一貫性**: 「明細一覧 (N件)」「レポート詳細」等の既存見出しと語感が揃うか

## 対応スコープ（決定後の想定）

1. 実装: `ReportInfoCard.tsx:83-89` の `<Paper>` 内先頭に `<Typography variant="subtitle2">` 等で見出しを追加
2. 設計書改訂:
   - `50_detail_design/screens/report-detail.md:56` の ASCII 区切り風記述を明示的な見出し記述に置換（または L88-114 §3 に見出し項目を追記）
   - `55_ui_component/screens/report-detail.md §ReportWorkflowInfo` に見出し Props の追加（または固定文言として責務欄に記載）
3. テスト: ReportInfoCard.test.tsx にセクション見出しのアサーション追加
