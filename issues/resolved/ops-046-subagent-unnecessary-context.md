# サブエージェントが指揮役専用のコンテキストを読み込んでいる

## 発見日
2026-03-29

## カテゴリ
performance

## 影響度
中

## 発見経緯
セッション内の議論で発見

## 関連ステップ
全 Step

## 問題

`.claude/rules/workflow.md` に `paths:` 制限がなく、全サブエージェントに読み込まれている。

workflow.md の内容の大半は指揮役専用：
- セッション管理（開始時・終了時の手順）
- コミットルール
- 品質ゲート判定基準
- 意思決定権限
- 成果物作成フロー

サブエージェント（designer, backend-developer 等）はこれらを読む必要がない。約2.4kトークンのコンテキスト圧迫。

## 提案

workflow.md に `paths:` 制限を追加してサブエージェントへの読み込みを制御するか、指揮役専用ルールを分離する。ただし `.claude/rules/` の paths 制限がサブエージェント起動時にどう適用されるかの検証が先に必要。

---

## 解決内容

`.claude/rules/workflow.md` を2つに分割し、サブエージェントへの不要なコンテキスト読み込みを解消した。

1. `.claude/rules/project-rules.md`（新規）: 全員に自動読み込みされる共通ルール（~23行）
2. `ai-dev-framework/guide/workflow.md`（新規）: 指揮役がセッション開始時に明示的に読むワークフロー手順
3. `.claude/rules/workflow.md` を削除
4. 参照元3ファイル4箇所を更新

サブエージェントのコンテキスト負荷: ~2.4kトークン → ~500トークンに削減。
