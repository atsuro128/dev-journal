### 判断・決定事項
- **マージ順序の原則**: 同階層の並列 PR は「スコープ小→大」で順次マージ、各マージ後に次 PR で master 取り込み（10-D → 10-C → 10-B BE → 10-B FE）
- **PR レビュー手順改善**: codex のスコープ拡大を防ぐため、チケットの「責務」「入力」を正本としてレビュースコープを確定。再レビューは前回指摘の解消確認 + 修正差分起因の回帰のみを blocker、それ以外は COMMENT に格下げ
- **押し返し 2 回ルール**: codex への押し返しを 2 回試して通らない場合は受け入れた方が効率的（レビュー往復コスト > 1 行修正コスト）。レート制限・workflow スタブ戻しの両ケースで実証
- **expense_date 対処方針**: 共有 DTO（`time.Time`）は変更せず、handler 層に `itemResponse` 構造体を用意して YYYY-MM-DD 文字列に変換
- **workflow 一覧の扱い**: ListPendingReports / ListPayableReports は 10-B から 501 スタブに戻し、本実装は 10-F スコープに分離（「実装したなら API 契約を満たす、defer ならスタブに戻す」原則）
- **指示書 ↔ issue の片方向参照**: issue 側から指示書を参照するのは可、指示書側に issue 番号を記載するのは不可
- **ops-058 を 10-X に統合**: `continue-on-error: true` 除去で FE テスト失敗が露呈、ops-058 単独では CI PASS しないため 10-X の横断品質確認と合わせて対応する方針に変更
