### 判断・決定事項

- **RS256 維持（ADR-0006）**: EdDSA は RS256 と同じく量子耐性なし、ML-DSA は Go エコシステム未成熟。NIST 2030 年非推奨は予防的ガイドラインであり移行根拠としては弱い。issue 050 で提起された懸念は ADR-0006 に「本システムが JWT を使う理由と判断ポイント」として記録
- **秘密鍵検証は os.Stat + TODO**: codex から PEM パース指摘を受けたが、Signer 未実装の現状では os.Stat で存在確認のみとし、Step 10 の NewSigner 実装時に PEM パース検証へ統合する方が責務として自然
- **LSP 統合は見送り（issue 036 revert）**: サブエージェントには LSP ツールが渡らない仕様制約。tools フィールドに LSP を追加してもサブエージェントの利用可能ツールは 6 個に固定され、CLAUDE.md やルール強化でも効果なし。Claude Code 側の対応を待って再検討
- **ADR は issue の解決記録ではない**: 背景セクションは「issue 050 で提起された」ではなくプロジェクト文脈（なぜこの判断が必要か）で書く。issue はきっかけに過ぎず、ADR 内で issue に紐づけない
- **codex の指摘を鵜呑みにしない**: 秘密鍵 PEM パース指摘は妥当だが、文脈を踏まえて os.Stat + TODO が最適解と判断し、codex に追加判断を求めて合意を得た
- **progress.md の細かい追記はノイズ**: ADR 追加など細粒度の変更は progress.md に書かない
