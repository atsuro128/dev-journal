## 1. 責務境界

### 含む観点

- 業務要件の妥当性（各 UC が実業務で達成可能か）
- ロール別の主要業務フロー（提出 → 承認 → 支払の一気通貫）
- UX の妥当性（操作感・用語・画面の分かりやすさ・期待通りに動くか）
- 自分のレポート操作（Approver/Accounting/Admin が申請者として操作する場合を含む）
- 異常系の業務上の許容度（却下理由表示、再申請のしやすさ）

### 含まない観点

- 個別エンドポイントの API 仕様検証（BE 自動テストで担保）
- 表示崩れ・文言の自然さ（`smoke_check.md` の領域）
- ロジックの境界値・例外網羅（自動テストで担保）

## 2. 重複排除の根拠

`traceability.md` の自動テスト反映先が埋まっている UC は機械的検証で正常系・異常系がカバーされている。本リストでは、UC が「業務の流れとして自然か」「実ユーザーの言葉で達成できるか」の人的判断を補完する観点に絞る。

| 区分 | 抽出方針 | 対応するチェックID |
|------|---------|-------------------|
| 業務シナリオ妥当性 | 各 UC を業務の文脈で実行し、全体の達成可否を判定 | UAT-001〜UAT-023 |
| ロール別フロー | 4 ロールの主要フローを一気通貫で実行 | UAT-030〜UAT-035 |
| UX 妥当性 | 用語の分かりやすさ、操作感、迷いポイント | UAT-040〜UAT-046 |
