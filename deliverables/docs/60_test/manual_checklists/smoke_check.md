## 1. 責務境界

### 含む観点

- ブラウザ実挙動（クリック挙動、二重押下防止、フォーカス制御）
- 表示崩れ・レスポンシブ（PC / スマホ幅）
- 文言の自然さ（日本語ラベル・トースト・エラーメッセージ）
- 体感導線（ロール別の入口画面、403 時の挙動、非表示要素）
- HTTP エラー時のフロント表示（401/403/404/409/422 のトースト・フォームエラー）
- 添付ファイルのブラウザ実挙動（5MB 境界、MIME 判定、ダウンロード起動）
- タイムスタンプ表示（JST 整合）
- ページネーション・フィルタの操作感
- キャッシュ／再読込整合性

### 含まない観点（自動テスト or UAT で担保）

- ロジック検証・状態遷移検証 → BE / FE 自動テストでカバー（traceability.md 参照）
- 業務フロー成立性（UC-* の達成可否） → E2E 自動テスト + `uat_check.md` でカバー
- データの永続化・テナント分離の確認 → BE 自動テスト（cross-cutting）でカバー
- 認可ロジック（ロール × リソース） → BE 自動テスト + RBAC マトリクスでカバー

| FE 側 `-`（補強） | NFR-UX-001（レスポンシブ）—自動 E2E は主要フロー検証のため、画面崩れの目視確認は手動が必須 | SMK-060〜SMK-063 |

NFR-DATA-006（UTF-8）は DB 設定で保証されるためマイグレーション手順書の責務とし、本リストでは扱わない。
