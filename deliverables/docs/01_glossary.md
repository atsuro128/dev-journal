# 用語集 (Glossary)

## この文書の役割

| 項目 | 内容 |
|------|------|
| 目的 | 全成果物で使う用語を統一する |
| 正本情報 | 用語定義、禁止表現、日英対応 |
| 扱わない内容 | 業務ルール、設計判断 |
| 主な参照先 | 全 Step の成果物 |

本プロジェクトで使用する用語の定義。ドキュメント・コード・コミュニケーションで統一して使用する。

## テナント・組織

| 用語 | 英語 | 定義 |
|---|---|---|
| テナント | Tenant | サービスを利用する1つの企業・組織。DB上は `tenants` テーブル、カラムは `tenant_id` で統一 |
| メンバー | Member (membership) | テナントに所属するユーザー。`tenant_memberships` テーブルで関連を管理 |
| 招待 | Invitation | Admin がメールアドレスとロールを指定してテナントへ招待する仕組み |

## ロール (RBAC)

| 用語 | 定義 |
|---|---|
| Admin | テナントの管理者。会社設定・メンバー管理・全データ操作・招待発行が可能 |
| Approver | 経費の承認者。経費承認・却下、自担当分の閲覧が可能 |
| Member | 一般メンバー。経費申請・編集（自分のみ）、状態確認が可能 |
| Accounting | 経理担当。経費一覧・エクスポート、支払済み更新が可能 |

## 経費精算

| 用語 | 英語 | 定義 |
|---|---|---|
| 経費レポート | Expense Report | 経費申請の単位。タイトル・対象期間を持ち、複数の経費明細を含む |
| 経費明細 | Expense Item | 経費レポート内の1件の支出。日付・金額・カテゴリ・摘要を持つ |
| 領収書 | Receipt / Attachment | 経費明細に紐づく証票ファイル（JPEG/PNG/PDF、5MB以下） |
| カテゴリ | Category | 経費の分類（交通費・宿泊費・飲食費・消耗品費・通信費・その他） |

## 状態遷移 (Workflow)

| 状態 | 英語 | 定義 |
|---|---|---|
| 下書き | draft | 作成中。編集・明細追加・削除が可能 |
| 提出済み | submitted | 承認者へ提出された状態。申請者による編集不可 |
| 承認済み | approved | 承認者が承認した状態。経理の支払処理待ち |
| 却下 | rejected | 承認者が却下した状態。却下理由が必須。再申請は新規レポートとして作成 |
| 支払済み | paid | 経理が支払処理完了を記録した最終状態 |

## 操作

| 用語 | 英語 | 定義 | 使わない表現 |
|---|---|---|---|
| 提出 | Submit | Member が下書きのレポートを承認者に提出する操作（draft → submitted） | 申請、送信 |
| 承認 | Approve | Approver が提出されたレポートを承認する操作（submitted → approved） | — |
| 却下 | Reject | Approver が提出されたレポートを差し戻す操作（submitted → rejected）。却下理由必須 | 差戻し、リジェクト |
| 支払完了 | Mark as Paid | Accounting が承認済みレポートの支払完了を記録する操作（approved → paid） | 精算、清算 |
| 再申請 | Resubmit | 却下後に新規レポートとして再度申請すること。元レポートへの参照を保持 | 再提出、やり直し |

## 技術用語

| 用語 | 定義 |
|---|---|
| RLS | Row Level Security。PostgreSQL の行レベルセキュリティ機能。テナント分離のDB層保証として使用 |
| JWT | JSON Web Token。RS256 アルゴリズムで署名。アクセストークン（15分）＋リフレッシュトークン（7日） |
| Argon2id | パスワードハッシュアルゴリズム。本プロジェクトの標準 |
| DB アクセスライブラリ | データベースアクセスライブラリ。具体的なツールは Step 3 で選定（sqlc / pgx 等） |
| 署名付きURL | Presigned URL。S3 上のファイルに一時的なアクセスを許可する URL。発行前に認可チェック必須 |
| 監査ログ | Audit Log。誰が・いつ・何を変更したかの記録。INSERT ONLY（更新・削除不可） |
| オフセットベースページネーション | Offset-based Pagination。`?page=1&per_page=20` 形式のページネーション |
| ADR | Architecture Decision Record。技術選定の判断理由を記録するドキュメント形式 |
