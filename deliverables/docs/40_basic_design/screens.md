## 1. 概要

本書は、経費精算SaaS MVP の全画面を一覧化し、画面IDの命名規則と共通UIパターンを定義する。
各画面の入力項目・バリデーション等の詳細は Step 5（詳細設計）で `screens/*.md` に定義する。

## 2. 画面ID命名規則

### 体系

```
SCR-{カテゴリ}-{連番}
```

| カテゴリ | 対象 | 例 |
|---------|------|-----|
| AUTH | 認証系（ログイン・サインアップ・パスワードリセット） | SCR-AUTH-001 |
| DASH | ダッシュボード | SCR-DASH-001 |
| RPT | 経費レポート系（一覧・作成・編集・詳細） | SCR-RPT-001 |
| WFL | ワークフロー系（承認待ち・支払待ち） | SCR-WFL-001 |
| ADM | 管理系（テナント情報・全レポート一覧） | SCR-ADM-001 |

### ルール

- 連番は3桁ゼロ埋め（001, 002, ...）
- モーダル・ダイアログは親画面のIDに `-D` サフィックスを付与しない。共通UIパターン（4章）として定義する
- Phase 3 で追加する画面には連番の空き番号を使用する

## 4. 共通UIパターン

### 4.1 レイアウト構成

認証済み画面は共通レイアウト（ヘッダー + サイドナビゲーション + メインコンテンツ）を使用する。
未認証画面（SCR-AUTH-*）はヘッダー・サイドナビゲーションを表示しない。

```
┌─────────────────────────────────────────────┐
│ ヘッダー（ロゴ / ユーザーメニュー）             │
├──────────┬──────────────────────────────────┤
│          │                                  │
│  サイド   │      メインコンテンツ              │
│  ナビ     │                                  │
│          │                                  │
│          │                                  │
└──────────┴──────────────────────────────────┘
```

レスポンシブ対応:
- PC (1024px以上): サイドナビゲーション常時表示
- タブレット (768px-1023px): サイドナビゲーションをハンバーガーメニューで開閉
- スマートフォン (767px以下): サイドナビゲーションをハンバーガーメニューで開閉、メインコンテンツは全幅

### 4.2 ヘッダー

| 要素 | 説明 |
|------|------|
| ロゴ | アプリケーション名。クリックでダッシュボードに遷移 |
| ユーザーメニュー | ユーザー名とロール名を表示。ドロップダウンで「ログアウト」を提供（対応UC: UC-SYS03） |

### 4.3 サイドナビゲーション

ロールに応じて表示するメニュー項目を制御する。

| メニュー項目 | 遷移先 | Member | Approver | Accounting | Admin |
|-------------|--------|--------|----------|------------|-------|
| ダッシュボード | SCR-DASH-001 | ○ | ○ | ○ | ○ |
| マイレポート | SCR-RPT-001 | ○ | ○ | ○ | ○ |
| レポート作成 | SCR-RPT-002 | ○ | ○ | ○ | ○ |
| 承認待ち | SCR-WFL-001 | - | ○ | - | - |
| 支払待ち | SCR-WFL-002 | - | - | ○ | - |
| 全レポート | SCR-ADM-001 | - | - | ○ | ○ |
| テナント情報 | SCR-ADM-002 | - | - | - | ○ |

> Accounting は Member としての経費申請も行うため「マイレポート」「レポート作成」を表示する（policies.md SS3.8 権限マトリクス準拠）。

### 4.4 エラー表示

#### フォームバリデーションエラー

- フィールドレベル: 各入力フィールドの直下に赤字でエラーメッセージを表示
- クライアントサイドバリデーション: 入力時またはフォーカスアウト時にリアルタイムで検証
- サーバーサイドエラー: APIレスポンスのエラー内容をフィールドにマッピングして表示

#### サーバーエラー

- API通信エラー（500系）: 画面上部にトーストでエラーメッセージを表示
- メッセージ例: 「サーバーとの通信に失敗しました。しばらくしてから再度お試しください。」

#### 認可エラー

- 403 Forbidden: 「この操作を行う権限がありません。」を画面上部にトーストで表示
- 401 Unauthorized: ログイン画面にリダイレクト

#### 404 Not Found

- 存在しないURLにアクセスした場合: 「ページが見つかりません。」をメインコンテンツ領域に表示
- APIレスポンスで404を受けた場合: 「指定されたデータが見つかりません。」をメインコンテンツ領域に表示

### 4.5 ローディング

- API通信中はメインコンテンツ領域にスケルトンUIを表示する
- 一覧画面: テーブル行のスケルトン
- 詳細画面: カード要素のスケルトン
- ボタン操作（提出・承認・却下・支払完了）: ボタンをdisabledにし、スピナーを表示

### 4.6 確認ダイアログ

以下の操作は確認ダイアログを表示してからAPIを実行する。

| 操作 | ダイアログメッセージ | 確認ボタン | 対応UC |
|------|-------------------|-----------|--------|
| レポート提出 | 「提出後は編集できなくなります。提出しますか?」 | 「提出する」 | UC-M06 |
| レポート削除 | 「このレポートを削除しますか? この操作は取り消せません。」 | 「削除する」 | UC-M07 |
| 明細削除 | 「この明細を削除しますか?」 | 「削除する」 | UC-M05 |
| 添付削除 | 「この添付ファイルを削除しますか?」 | 「削除する」 | UC-M03a |
| 承認 | 「このレポートを承認しますか?」（承認コメント入力フォーム付き。任意、0〜1000文字） | 「承認する」 | UC-A02 |
| 却下 | 「このレポートを却下しますか?」（却下理由入力フォーム付き。必須、1〜1000文字） | 「却下する」 | UC-A03 |
| 支払完了 | 「このレポートの支払完了を記録しますか?」 | 「支払完了にする」 | UC-AC02 |

### 4.7 空状態

データが存在しない場合の表示。

| 画面 | 空状態メッセージ |
|------|----------------|
| レポート一覧（自分） | 「経費レポートはまだありません。レポートを作成して経費精算を始めましょう。」 |
| 承認待ち一覧 | 「承認待ちのレポートはありません。」 |
| 支払待ち一覧 | 「支払待ちのレポートはありません。」 |
| テナント全レポート一覧 | 「レポートはまだ作成されていません。」 |
| レポート詳細の明細一覧 | 「明細はまだ追加されていません。「明細追加」から経費を登録してください。」 |

### 4.8 ステータスバッジ

レポートのステータスは色分けしたバッジで表示する。

| ステータス | 日本語表記 | バッジ色 |
|-----------|-----------|---------|
| draft | 下書き | グレー |
| submitted | 提出済み | 青 |
| approved | 承認済み | 緑 |
| rejected | 却下 | 赤 |
| paid | 支払済み | 紫 |

### 4.9 ページネーション

- オフセットベースページネーション（デフォルト20件/ページ）
- 一覧画面の末尾にページネーションコントロール（ページ番号 + 前へ/次へボタン）を配置
- 現在のページ番号をハイライト表示する
- 総ページ数が多い場合は省略表示する（例: 1 2 3 ... 8 9 10）
