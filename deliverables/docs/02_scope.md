## MVP の一言定義

> マルチテナント環境で、経費レポートの作成・提出・承認・却下・支払完了の一連のワークフローを、4ロールのRBACで制御し、テナント分離を二重保証（アプリ層 + RLS）するWebアプリケーション。

## MVP（必須スコープ）

### 認証・ユーザー管理
- サインアップ / ログイン（メール + パスワード）
- JWT 認証（アクセストークン 15分、リフレッシュトークン 7日）
- パスワードハッシュ: Argon2id
- ログアウト（リフレッシュトークン無効化）
- パスワードリセット（リセットトークン経由）

### RBAC
- 4ロール: Admin / Approver / Member / Accounting
- 全 API でミドルウェアによるロール検証
- リソース所有者チェック（Member は自分の経費のみ操作可能）

### テナント分離
- 全テーブル・全クエリで `tenant_id` 必須
- リポジトリ層での `tenant_id` 強制（ハンドラで直接SQL禁止）
- PostgreSQL RLS による二重保証

### 経費レポート
- レポート CRUD（タイトル・対象期間）
- 経費明細 CRUD（日付・金額・カテゴリ・摘要）
- 6つの標準カテゴリ（交通費・宿泊費・飲食費・消耗品費・通信費・その他）

### 承認フロー（状態遷移）
- draft → submitted → approved → paid
- submitted → rejected
- 不正な遷移のドメイン層での拒否
- 却下理由必須、承認コメント任意
- 却下後の再申請（新規レポートとして作成、元レポートへの参照保持）

### 添付ファイル
- 領収書アップロード（S3）
- ファイルタイプ制限: JPEG, PNG, PDF
- サイズ制限: 5MB/ファイル
- 署名付きURL発行（発行前の認可チェック必須）

### フロントエンド
- ログイン画面
- ダッシュボード（経費サマリー）
- 経費レポート一覧・詳細・作成・編集
- 承認操作画面（Approver用）

### テスト
- テナント分離テスト
- RBAC テスト
- 状態遷移テスト
- ローカルテスト実行（ci.yml は参考用デモとしてリポジトリに残す）

### インフラ
- Docker Compose（ローカル開発）
- AWS デプロイ（ECS Fargate / RDS / S3）
- ローカルテスト（lint / test / build）+ 手動デプロイ（GHA の ci.yml / deploy.yml は参考用デモとして残す）
- ヘルスチェックエンドポイント（`/health`）

## Phase 3 追加機能（MVPの次）

| 機能 | 概要 | 優先度 |
|---|---|---|
| 招待フロー | Admin がメール+ロール指定で招待。トークン（72時間有効）経由のアカウント作成 | 高 |
| アプリ内通知 | 提出・承認・却下・支払完了のイベント通知。既読管理 | 高 |
| 監査ログ | 操作履歴の記録（INSERT ONLY）。tenant_id / user_id / action / 変更前後の値 | 高 |
| CSV エクスポート | 経費一覧のCSV出力。Accounting / Admin のみ。期間・ステータス等でフィルタ | 中 |
| メンバー管理画面 | メンバー一覧・ロール変更・削除（Admin のみ） | 中 |
| テナント設定画面 | 会社情報編集（Admin のみ） | 低 |

---

## やらないこと（スコープ外）

| 項目 | 理由 |
|---|---|
| メール通知 | MVP ではアプリ内通知のみ。メール送信は外部サービス連携のコストが高い |
| 多段階承認 | MVP では1段階承認（Approver 1名）。承認ルートの設計は複雑度が高い |
| カスタムカテゴリ | テナントごとのカテゴリ定義は将来拡張可能な設計にするが、MVP では固定6カテゴリ |
| 多言語対応 (i18n) | UI は日本語固定。README・API ドキュメントのみ英語対応 |
| モバイルアプリ | Web アプリのみ。レスポンシブ対応はするがネイティブアプリは作らない |
| OCR / 領収書自動読み取り | 手動入力のみ |
| 外部会計ソフト連携 | API 連携（freee, MF等）は対象外 |
| SSO / OAuth ログイン | メール + パスワード認証のみ |
| ファイルのウイルスチェック | S3アップロード時のスキャンは対象外 |
| データのバックアップ・リストア手順 | RDS の自動バックアップに委ねる。手動リストア手順は文書化しない |
